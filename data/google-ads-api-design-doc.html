Google Ads API — Application Design Document

Prepared by: Eitan Yariv, WAO Digital Marketing Agency Date: July 2026 App URL: https://www.wao.co.il/google-ads/onboarding

1. Executive Summary

WAO is an Israeli digital marketing agency managing Google Ads campaigns for small and medium-sized B2C businesses. The WAO Automation Platform is an internal agency tool that streamlines the end-to-end onboarding of new clients onto Google Ads — from initial intake through campaign launch — using the Google Ads API.

The platform operates exclusively under WAO's existing MCC (Manager Account). It does not expose Google Ads data or controls to end clients; all campaign management remains in WAO's hands as the authorized agency.

2. Application Overview

2.1 App Name

WAO Automation Platform

2.2 App URL

https://www.wao.co.il/google-ads/onboarding

2.3 Purpose

The platform automates three operations that WAO currently performs manually for each new client:

  1. Creating a new Google Ads sub-account linked to WAO's MCC
  2. Creating conversion actions (form submission, phone call, WhatsApp click)
  3. Creating and publishing the initial Search campaign with a Responsive Search Ad

These three steps currently take 45–90 minutes per client when done manually in the Google Ads UI. The API integration reduces this to under 2 minutes, allowing WAO to scale from 10 to 100+ active clients without proportionally increasing operational headcount.

2.4 Users

Internal use only. The platform is operated exclusively by WAO staff and the WAO automated pipeline. No end clients have access to the platform or to the Google Ads API credentials.

3. Google Ads API Usage

3.1 API Operations Used

OperationAPI Service / MethodPurpose
Create client sub-account CustomerService.CreateCustomerClient Creates a new managed Google Ads account under WAO's MCC for each new client. Currency: ILS, timezone: Asia/Jerusalem.
Create conversion actions ConversionActionService.MutateConversionActions Creates three conversion actions per account: form lead, phone call, WhatsApp click. Values are set based on client's average job value collected during onboarding.
Get conversion action tags ConversionActionService.GetConversionAction Retrieves the gtag snippet and event snippet for embedding in the client's landing page.
Create campaign budget CampaignBudgetService.MutateCampaignBudgets Creates a shared budget using the client's agreed monthly spend divided by 30.
Create Search campaign CampaignService.MutateCampaigns Creates a Search campaign with Maximize Clicks bidding, Hebrew language targeting, and geo targeting to the client's service cities.
Create ad group AdGroupService.MutateAdGroups Creates a single ad group per campaign during the initial launch phase.
Create keywords AdGroupCriterionService.MutateAdGroupCriteria Adds Hebrew-language keyword list (generated by WAO's strategy AI) as Exact and Phrase match. Adds negative keywords to filter irrelevant traffic.
Create Responsive Search Ad AdGroupAdService.MutateAdGroupAds Creates an RSA using headlines and descriptions generated by WAO's copy AI (3 headlines minimum, 2 descriptions).
Add geo targets AdGroupCriterionService.MutateAdGroupCriteria (geo) Targets the specific Israeli cities provided by the client during onboarding.

3.2 Account Hierarchy

WAO MCC (Manager Account) → Client Sub-Account (created per client) → Campaign → Ad Group → RSA + Keywords

All sub-accounts are created under WAO's single MCC. WAO retains manager access and billing control. The client is aware that WAO manages their Google Ads account on their behalf.

3.3 Data Flow

  1. Business owner completes WAO's onboarding chatbot (collects: business type, service cities, budget, contact info, USP, reviews).
  2. WAO's AI generates campaign strategy (keywords, bidding, geo targets) and ad copy (headlines, descriptions).
  3. Client reviews and approves the plan on-screen.
  4. Client pays the one-time setup fee (₪9.90 via Yaad Sarig payment gateway).
  5. Platform calls Google Ads API: creates sub-account → conversion actions → budget → campaign → ad group → keywords → RSA.
  6. Platform generates a static landing page and deploys it to Cloudflare Pages.
  7. Google Ads conversion tags (from step 5) are embedded in the deployed landing page.
  8. WAO team reviews the account before enabling the campaign.

5. Security & Compliance

5.1 Credential Handling

  • Google Ads API credentials (client ID, client secret, developer token, refresh token) are stored exclusively in server-side environment variables — never in client-side code or version control.
  • The refresh token is scoped to WAO's own Google account (MCC owner) only.
  • No end clients have access to WAO's API credentials or the automation platform.

5.2 Data Storage

  • Campaign configuration data (business name, phone, cities, budget) is stored on WAO's private server in a JSON file per client.
  • No Google Ads performance data (impressions, clicks, conversions) is stored externally — it remains in the Google Ads platform.
  • Client personal data is handled in accordance with Israel's Privacy Protection Law (חוק הגנת הפרטיות).

5.3 Client Consent

Before campaign creation, clients explicitly accept WAO's terms of service, which include authorization for WAO to create and manage a Google Ads account on their behalf. A consent timestamp is recorded and stored with each campaign record.
